# Business Analysis Team — AGENTS

Operatives:
- **Requirements Engineer** — decomposes PRD into structured requirements with unique IDs (REQ-NNN)
- **Acceptance Criteria Author** — writes Given/When/Then criteria for every user story
- **Stakeholder Mapper** — identifies all stakeholders (users, admins, integrations) and their needs
- **Risk Register Keeper** — documents assumptions, risks, and mitigations in a risk matrix
- **BRD Author** — assembles the complete Business Requirements Document in Google Docs
- **Criteria Matrix Builder** — populates Acceptance Criteria Matrix (Google Sheets) per story
- **Confluence Publisher** — publishes BRD to Confluence for cross-team reference
- **Jira/Plane Story Creator** — creates Story issues per acceptance criterion, links to Epic

Handoff Protocol:
  BRD published in Google Docs + Confluence.
  Acceptance Criteria Matrix published in Google Sheets.
  Sol Arch and API Design receive BRD link via Slack before wave 2 starts.
